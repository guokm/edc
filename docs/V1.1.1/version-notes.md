# V1.1.1 运营登录与角色门户版

## 1. 本次迭代目标

V1.1.1 是在 V1.1 商用基座之上的一个小版本，重点不是继续加后端能力，而是把已经落好的账号、参与方、RBAC 真的接到前端页面上，让“谁登录、看到什么、代表谁操作”能够现场演示。

## 2. 本次完成内容

### 2.1 前端登录态接通

- 前端新增本地登录态管理：
  - 登录
  - 退出
  - 本地会话持久化
  - 会话过期时间展示
- 顶部头部改为“品牌区 + 登录状态区 + 角色入口 + 角色内导航”。
- 未登录时展示角色账号卡片，点击后可自动填充演示账号。

### 2.2 角色门户收敛

- 供应方账号只能进入供应方页面。
- 消费方账号只能进入消费方页面。
- 运营方账号只能进入运营方页面。
- 角色卡片未授权时置灰，避免现场演示时角色混淆。

### 2.3 运营主数据门户

- 运营方新增“运营主数据”页面。
- 页面直接展示并讲清三层关系：
  - `组织（organization）`
  - `参与方（participant）`
  - `账号（user account）`
- 用于向客户说明：
  - 组织是谁
  - `participantId` 在协商/目录/计费链路里代表谁
  - 登录账号具体代表哪个参与方执行动作

### 2.4 Operator 接口前端令牌自动注入

- `frontend/src/lib/http.ts` 新增 Operator Token Resolver。
- 前端调用 `/op/**` 接口时会自动带上 `X-Operator-Token`。
- 运营方页面不再需要手工拼令牌调用组织/参与方/账号列表接口。

## 4. 演示账号

- `provider_admin / ChangeMe@123`
- `consumer_admin / ChangeMe@123`
- `operator_admin / ChangeMe@123`

说明：这三个账号分别绑定 `participant-a`、`participant-b`、`operator`，用于演示角色门户切换，不建议在客户环境保留默认密码。

## 6. 下一步建议

下一刀原建议为资产生命周期与 Offer 套餐可视化；实际 V1.1.2 已优先落地“运行态可观测增强”，因此该方向顺延为 `V1.1.3 资产生命周期与 Offer 套餐可视化版`：

- 资产状态：草稿 / 已发布 / 已下架 / 已归档
- Offer 套餐卡片化展示
- 供应方页面补充“资产 -> Offer -> 协商 -> 协议”关系图
- 运营方页面补充“组织 -> 参与方 -> 会员 -> 凭证 -> 协商”追踪视图
