# V1.1.2 运行态可观测增强版

## 1. 本次迭代目标

V1.1.2 是在 V1.1.1 角色门户基础上的小版本，重点放在“演示和排障时能一眼看懂系统是否可用”。本次不扩展新的业务流程，优先增强节点与健康页面的运行态摘要、异常提示和双 Data Plane 承载信息。

## 2. 项目现状分析

- 后端已经形成 Maven 多模块架构，控制面、数据面、身份、发行、联邦目录、运营服务职责清晰。
- Docker Compose 与 `scripts/verify.sh` 已经能覆盖前端入口、Operator 登录、协商、传输、双数据面和计费校验。
- 前端已经完成角色门户收敛，供应方、消费方、运营方页面边界明确。
- 当前可继续提升的短板在可观测页面：原有页面保留了明细，但缺少顶部汇总和异常聚焦，现场演示时需要逐项扫描。

## 3. 本次完成内容

### 3.1 节点健康摘要

- 在“节点与健康”页面顶部新增四个运行态摘要：
  - 健康检查通过数
  - 异常项数量
  - 已注册 Data Plane 数量
  - 双数据面累计传输数
- 保留原健康检查卡片和分页明细，避免丢失排障信息。

### 3.2 异常项聚焦

- 当健康检查存在非 `UP` 项时，页面会在明细前展示异常清单。
- 异常清单展示模块、检查项、返回状态码和错误信息，方便现场快速定位服务问题。

### 3.3 Data Plane 状态表达优化

- 控制面注册表中的节点状态改为状态标签。
- 数据平面运行摘要卡片增加状态标签，并突出累计传输数量。

## 4. 变更文件

  - 版本标识更新为 `V1.1.2 · Ops Observability`。
