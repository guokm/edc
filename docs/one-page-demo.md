# 一眼演示文档（按角色拆分）

## 1. 角色入口与页面

| 角色 | 入口按钮 | 可操作页面 | 本角色不做 |
| --- | --- | --- | --- |
| 供应方（华东车联） | `供应方入口` | `供应方工作台`、`节点与健康`、`角色演示文档` | 不发起协商、不主动拉取数据 |
| 消费方（保险风控中心） | `消费方入口` | `消费方工作台`、`角色演示文档` | 不发布资产、不做治理巡检 |
| 运营方（数据空间运营平台） | `运营方入口` | `治理模块接口`、`节点与健康`、`角色演示文档` | 不参与资产协商与取数 |

## 2. 10 分钟演示脚本（公司宣讲）

### 2.1 供应方（3 分钟）

1. 进入 `供应方入口 -> 供应方工作台`。  
2. 在 `供应方资产发布` 先点击 `刷新策略列表` 或 `创建默认策略`。  
3. 选择策略ID后，填写资产信息，点击 `发布资产（供应方）`。  
4. 讲解点：右侧同步轨迹会显示“创建资产 -> 写入Offer -> 刷新目录”，说明“资产已上架可被发现”。

### 2.2 消费方（4 分钟）

1. 切换 `消费方入口 -> 消费方工作台`。  
2. 先确认运营侧存在 `participant-b` 的 `ACTIVE` 会员，且已完成签发资格（无会员/无签发资格时协商与传输返回 `403`）。  
3. 点击 `刷新目录` 并选中资产。  
4. 点击 `执行签发并签约`（Issuer签发 -> Identity入库 -> DCP展示/校验 -> 创建协商）。  
5. 在 `协商合同（协议）` 中点击 `用于传输`，再点击 `发起传输`。  
6. 在 `数据传输状态与链路追踪` 点击 `拉取数据`。  
7. 讲解点：从“看见资产”到“完成签发签约再取数”全流程闭环。

> 失败演示点：把 offerId 改成错误值再点协商，会返回失败，并在协商列表显示 `REJECTED_OFFER_NOT_FOUND`。

### 2.3 运营方（3 分钟）

1. 切换 `运营方入口 -> 治理模块接口`。  
2. 点击 `一键巡检`，展示 Identity / Issuer / Federated / Operator 统一结果。  
3. 点击 `额度校验`，展示按次计费返回 `used/remaining/estimatedAmount`。  
4. 切到 `节点与健康`，展示双数据平面与整体健康状态。

## 3. 讲解主线（1 句话版本）

> 供应方先发布资产，消费方完成协商签约并拉取数据，运营方最后证明治理、计费和节点稳定性，三方职责清晰且可追踪。
